Все задачи реализуются в анонимных блоках.


1. Отловить 2 стандартных исключения при поиске клиента по ИНН (NO_DATA_FOUND, TOO_MANY_ROWS),в секцию output вывести 
корректное описание ошибки (понятное для пользователя). 
Все остальные исключения выводить как "Непредвиденная ошибка"

2. При поиске клиента по Фамилии (where name like '%Иванов%') отловить исключение возврата более 1й записи
с помощью команды RAISE (придумать имя исключению и обработать в секции exception).

3. При вставке информации по клиенту, отловить момент когда ИНН пустое поле (переменная с инн = null),
с помощью команды RAISE_APPLICATION_ERROR выбросить исключение, отловить его в секции Exception и отобразить
понятное для пользователя описание ошибки в output
